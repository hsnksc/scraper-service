# 06.04.2026 Gelistirme Notlari

## 1. Projenin Amaci
Bu proje, verilen koordinata gore emlak ilanlarini bulup:
- `full scrape` modunda (Playwright ile sayfaya girip detay cekerek),
- `discovery-only` modunda (arama API'lerinden hizli aday toplayarak),
sonuclari API response olarak dondurur ve istenirse dis sisteme post eder.

## 2. Calisma Mantigi (Mimari Ozet)
Iki temel akış var:

1. `Scrape` akisi (`/api/scrape`, `/api/scrape/run`, `/api/scrape/stream`)
- `scraper/orchestrator.py` icindeki `run_scrape` calisir.
- Koordinat reverse geocode edilir, lokal query'ler uretilir.
- `discover_listing_urls` ile aday URL'ler bulunur.
- Site bazli scraper (Sahibinden/Hepsiemlak/Emlakjet/Hurriyetemlak) veya generic scraper ile detay cekilir.
- Sonuclar normalize edilir, filtrelenir, dengelenir ve dis API'ye post edilir.

2. `Discovery-only` hizli akis (`/api/discovery/run`)
- Playwright scrape yok; sadece arama API'lerinden adaylar toplanir.
- `main.py` -> `discover_listing_urls(..., fast_mode=True)` cagrisi yapar.
- Adaylardan pseudo listing olusturulur.
- `response_enrichment.py` ile:
  - kategoriler,
  - turetilmis metrikler,
  - `market_rows` (cins + fiyat + m2 odakli satirlar)
  uretilir.

### 3.2 Fast discovery tuning
- Provider query limiti env ile yonetilebilir hale geldi.
- Provider basina max sonuc env ile yonetilebilir hale geldi.
- Eklenen/duzenlenen env ayarlari:
  - `DISCOVERY_PROVIDER_QUERY_LIMIT=3`
  - `DISCOVERY_MAX_RESULTS_PER_PROVIDER=10`
  - `DISCOVERY_PROVIDER_TIMEOUT_SECONDS=8`

### 3.3 API seviyesinde strict filtre override
- `/api/discovery/run` artik request body'den su flag'leri aliyor:
  - `strict_detail_only` (opsiyonel bool)
  - `strict_district_header` (opsiyonel bool)
- Boylece her sorguda filtre sertligi request bazinda degistirilebiliyor.
- Ilgili dosyalar:
  - `main.py`
  - `scraper/google_search.py` (`_rank_and_filter_candidates`, `_locality_score`, `discover_listing_urls`)

### 3.4 Fiyat/m2 parser iyilestirmeleri
- `TL` formatlarini daha saglikli parse eden yeni sayi parser eklendi.
- Binlik/ondalik ayraclarinda yanlis buyutme etkisi azaltildi.
- Fiyat, m2 ve oda cikarimi once `title`, sonra `snippet` olacak sekilde iyilestirildi.
- Ilgili dosya:
  - `scraper/google_search.py`

### 3.5 Cikti formati: market_rows
- Istek odagina uygun olarak `cins + fiyat + m2` tablosu eklendi.
- `m2` yoksa kayit yine listede kalir (sadece fiyat olanlar da dahil).
- Alanlar:
  - `listing_type`
  - `property_type`
  - `listing_kind`
  - `price_try`
  - `size_m2_try`
  - `price_per_m2_try`
  - `title`
  - `source_url`
- Ilgili dosya:
  - `scraper/response_enrichment.py`
  - `run_discovery_fast.ps1` (hizli script cikti testi icin)
- Yeni env:
  - `MARKET_ROWS_LIMIT=80`

## 4. Teknik Akis (Discovery)
1. `reverse_geocode(lat,lng)` ile mahalle/ilce/sehir/yol bilgisi alinir.
2. Koordinata ozel lokal query'ler uretilir.
3. API kaynaklari paralel calisir:
- Tavily
- Exa
- Serper
4. Sonuclar URL bazinda birlestirilir, duplicate temizlenir.
5. Locality score ile lokal iliskisi dusuk adaylar elenir.
6. Istenirse detail URL agirlikli filtre uygulanir.
7. Pseudo listing -> enrichment -> `market_rows` uretilir.

## 5. Ornek Kullanim

### 5.1 Hızlı script
```powershell
.\run_discovery_fast.ps1 -Lat 36.85762999247417 -Lng 30.787314375010823 -MaxPerSource 12 -QueryLimit 4 -StrictDetailOnly $false
```

### 5.2 API
```powershell
$body='{"lat":"36.85762999247417","lng":"30.787314375010823","listing_type":"all","property_type":"all","num_pages":1,"strict_detail_only":false,"strict_district_header":true}'
Invoke-RestMethod -Method Post -Uri "http://127.0.0.1:8010/api/discovery/run" -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 8
```

## 6. Cikti Okuma Notu
- Ana odak alan: `market_rows`
- Pratikte kullanilacak alanlar:
  - `listing_type` + `property_type` => cins
  - `price_try` => fiyat
  - `size_m2_try` => m2
  - `price_per_m2_try` => m2 fiyat (varsa)
- `size_m2_try` null olabilir; bu kayitlar "fiyat var, m2 yok" olarak yine de korunur.
